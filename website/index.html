Core Capabilities

Engineered for anonymity.
Built for compliance.

Every design decision in OpenWhistle prioritises the safety of the whistleblower and the legal obligations of the organisation.

100% Anonymous

Four-layer IP anonymisation: nginx strips X-Forwarded-For, the application middleware drops the remote address, no IP column exists in the database schema, and Redis sessions carry no identifying metadata.

  • No IP in DB
  • No fingerprinting
  • UUID4 PIN
  • Anon sessions

HinSchG Compliant

Tracks §17 SLA deadlines: 7-day acknowledgement and 3-month feedback timers. Bidirectional messaging between whistleblower and case handler via the same secure channel, as required by §17 Abs. 3.

  • §17 SLA timers
  • Bidirectional
  • §26 Data retention
  • Hard deletion

Secure Admin Portal

Mandatory TOTP-based multi-factor authentication for every admin account. OIDC login support for enterprise SSO. Web-based first-run setup wizard. Rate-limited without IP tracking.

  • TOTP MFA
  • OIDC / SSO
  • Setup wizard
  • Redis rate limit

Fully Self-Hosted

Docker Compose deployment. Your data stays on your infrastructure — no external CDN calls, no telemetry, no cloud dependency. PostgreSQL + Redis. Fonts served locally to avoid DSGVO CDN violations (LG München 2022).

  • Docker Compose
  • PostgreSQL
  • Redis
  • No telemetry
Legal Context

Who is required to act?

HinSchG §12 Obligation to establish internal reporting channels

The Hinweisgeberschutzgesetz, in force since 2 July 2023, implements EU Directive 2019/1937 into German law. It mandates secure, anonymous internal reporting systems for:

  • Private-sector companies with 50 or more employees
  • All federal and state public authorities
  • Municipalities with a population of 10,000 or more
  • Financial sector firms regardless of employee count
  • Aviation, maritime, and nuclear sector operators
HinSchG §8 Strict confidentiality of identity

The whistleblower's identity must be protected at all times. No metadata that could directly or indirectly identify them may be stored — including IP addresses, browser fingerprints, or session tokens linked to identity.

Violation can result in criminal sanctions under §39 HinSchG.

Key Deadlines — §17
Obligation Deadline
Acknowledge receipt of report 7 days
Provide feedback on measures taken 3 months
Delete data after case closure 3 years
How OpenWhistle helps

The admin dashboard tracks all §17 SLA deadlines with colour-coded warnings. The bidirectional message thread is the legally mandated secure channel (§17 Abs. 3). §26 data retention is enforced through scheduled case closure and hard deletion.

Quick Start

Up and running in minutes.

OpenWhistle is distributed as a Docker image published to GitHub Container Registry, Docker Hub, and quay.io. A single docker compose up -d starts the application, PostgreSQL, Redis, and the nginx reverse proxy.

After first boot, the /setup route launches a web wizard to create the first administrator account and configure MFA. The wizard is disabled automatically after completion.

Full installation guide →
bash — openwhistle installation
# Clone the repository
$ git clone https://github.com/openwhistle/OpenWhistle
$ cd OpenWhistle

# Copy and edit environment config
$ cp .env.example .env
$ nano .env

# Start all services
$ docker compose up -d
▶ Network openwhistle_default Created
▶ Container openwhistle-db-1 Started
▶ Container openwhistle-redis-1 Started
▶ Container openwhistle-app-1 Started
▶ Container openwhistle-nginx-1 Started

# Open the setup wizard
$ open http://localhost:4009/setup
